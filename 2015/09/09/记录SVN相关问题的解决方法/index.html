记录SVN相关问题的解决方法

2015-09-09 | lynulzy |

记录一下今天遇到的svn的问题，避免以后出现再浪费太多时间。

###问题的起因

The operation could not be performed because “PROJECTNAME” has one or more tree conflicts

关于这个问题我在StackOverflow上也回答过一次。

###SVN版本降级

问题描述

在Xcode中如果项目使用的svn版本太低会提示需要版本升级才能使用Xcode提供的svn版本控制服务，升级之后会将Mac自带的svn升级，而此时如果在终端对较新版本的svn将不能使用。–需要对系统的svn版本降级。

如果对于一个习惯使用 Git 的开发者而言，SVN 落后于时代是不言而喻的，但是当前的现状就是不得不坚持使用SVN而不是Git，所以就事论事，怎么将SVN版本降低到可用状态？

A：
我在网络上找了许多相关资料，并没有一种能够解决我所遇到的问题，直到发现这篇文章,在Mac中svn是默认配置，也即一台Mac上默认的会安装一个版本的svn。卸载？不好意思，卸载了会重装，而重装的这个版本也是你所使用是OS版本匹配的SVN版本。 我现在的SVN版本是10.10 默认配置 svn 1.7，恰好能够满足我的需求。
