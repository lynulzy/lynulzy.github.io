CocoaPods使用注意事项

  • CocoaPods

###问题的来源：

前言

地图库高德地图 SDK 包含一系列的SDK，有2D、3D地图类的、搜索类的、导航类的、云图类的。

在项目中原先使用的是 AMapSearch + AMap2D ，但是需要用到AMapNavi 中的路径规划功能，所以需要把冲突的库去掉，安装一个新的库，所以改变之后的库是 AMapSearch + AMapNavi 。

问题描述完毕。

###处理

使用过CocoaPods的朋友知道，在CocoaPods 中添加一个库就是在Podfile 中添加一条记录，并执行 pod install 命令即可。但是在当前（2015-09）CocoaPods 版本稳定版本下 0.38.2 下CocoaPods 集成 AMapSearch 库会出现如下报错

Errno::ENOENT - No such file or directory AMapSearch/AMapSearchKit.framework/Resources

在github上有相关的解决办法，并且有大神表示这是一个CocoaPods版本的bug，在之后的版本会消除。

总结两种解决办法

  1. 删除报错信息中的指定目录或者文件，然后执行 pod install即可；
  2. 手动删除 workspace 下 Pods 中相关的文件，然后执行pod install.

关于手动删除Pods中的三方库

pod install 中出现的错误信息，我们发现一个Pod的集成过程中有一步是向相关目录和文件中写入内容的。所以，删除引用可以从删除这些目录和文件中的三方库信息开始。

  1. Pods 目录下 Manifest.lock 文件中删除AMapSearch行
  2. Pods 目录下 删除 AMapSearch 目录，这个目录下是AMapSearch的framework文件。
  3. Pods 目录下 Headers-> Public 下删除 AMapSearch 目录及其文件
  4. Pods 目录下 Target Support Files 目录中 AMapSearch目录及其文件。

过程中可能会一并出现一些关于版本控制的错误，具体的表现是在Xcode中有 文件找不到，但实际上这些版本控制下的文件是与当前的工程无关的，清除的方法是在 Terminal 中进入相关文件的上层目录执行 *svn delete \.

延伸

今天在看 CocoaPods 的 issue的时候发现cocoaPods是有相关的三方库解除工具的，可以参考这个连接

具体来讲就是安装一个名字是 cocoapods-deintegrate的工具，并在工程目录中执行 pod deintegrate 命令。

而这个命令的作用则是移除项目中集成的所有 cocoapods 。

引用一下原文

A CocoaPods plugin to remove and deintegrate CocoaPods from your project. Removing all traces of CocoaPods from an Xcode project.
